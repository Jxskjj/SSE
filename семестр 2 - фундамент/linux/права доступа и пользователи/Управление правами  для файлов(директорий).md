В Linux у каждого файла и директории есть свои владельцы. Под владельцами подразумевается пользователь и группа.
Владельцы устаналиваются при создании файла или же директории. Пользователь, который создал файл автоматически становится его владельцем, и также группой файла становится группа, в которой состоит пользователь. 
При попытке доступа к файлу оболочка в три этапа проверяет, имеем ли мы доступ к файлу:
- проверка являемся ли мы владельцем
- если мы не являемся владельцем проверяется группа, которой принадлежит файл 
- если мы не подходим под оба условия то мы получаем права `Other` 

### Изменение владельца
Для изменения владельца используется команда `chown`, в которой мы указываем пользователя и файл, у которого мы меняем владельца на ранее указанного.
```bash
chown jxskj /home/accounts
```
В этой команде есть одна важная опция, которая встречается в большинстве команд. Опция `-R` позволяет рекурсивно задавать права доступа не только к самой директории но и к всему её содержимого.  Команда `chown -R jxksj /home/accounts`, уже не просто передаст права директории `accounts`, но и всем её вложенным. 

### Измнение владельца группы 
Для изменения группы можно применить команду `chgrp`, либо же также использовать `chown`, которая также способна изменять группы. Приведем примеры изменения группы:
```bash
chgrp .sales /home/accounts #стандартное изменение группы
chown jxskj.code main.c #изменяем владельца на jxskj а группу на code. Также вместо точки между именем и группой можно ставить :
chown .code main.c #изменение группы без изменнеия пользователя. Также можно использовать :
```
Как и в случае с `chown`, здесь применима опция `-R` для рекурсивного изменения.
Как писалось ранее, группой файла становится группа, в которой состоит пользователь. Но иногда он есть в нескольких группах и одна из них должна быть как "целевая". 
```bash
╭─ jxskj@thinkpad ~
╰─❯ groups
jxskj wheel
╭─ jxskj@thinkpad ~
╰─❯ newgrp wheel
╭─ jxskj@thinkpad ~
╰─❯ groups      
wheel jxskj
```
Выше приведен пример изменения целевой группы. Это правило будет работать до первой перезагрузки или до первого выхода из системы. Чтобы пользователь мог выдвинуть группу как целевую он ОБЯЗАТЕЛЬНО должен быть её членом. Иначе оболочка потребует ввести пароль группы(его можно установить через команду `gpasswd)

### Права в linux
В linux у нас есть 3 основных права: read, write, execute. Рассотрим на что они влияют в отношении файлов и диреткорий.

| Права   | файлы                        | директории                                     |
| ------- | ---------------------------- | ---------------------------------------------- |
| Read    | прочитать содержимого файла  | отобразить списка содержимого директории       |
| Write   | изменить содержание файла    | создавать, удалять или задавать права на файлы |
| Execute | запустить файл как программу | перейти в директорию                           |

Для изменения прав используется команда `chmod`, в которой мы задаем права на директорию или файл. Права можно задать как в буквенном, так и в числовом формате. По умолчанию права идут в следующем порядке: user -> group -> other.

Разеберем как установить права в численном  и буквенном значении:
- Read - `4` или `r`
- Write - `2` или `w`
- Execute - `1` или `x`
Когда мы задаем права в числовом формате мы записываем трехзначное число, в котором каждая цифра явлется суммой прав для той или иной группы. Например:
```bash
chmod 755 /somefile
```
Даст макисмальные права для текущего пользователя, чтение и запуск для группы и `other`. 
Если же мы использует буквенный формат, то мы используем `+` и `-` для доступа или запрета на определенные права. Привдем пример нескольких команд для предоставления прав. 
```bash
chmod +x /somefile #разрешения на запуск для всех
chmod go-rw secret.txt #запрет группе и остальным на чтение и запись
chmod a=r document.pdf #дать права всем только на чтение
chmod u=rwx,g=r file.txt # пользователю - всё, групепе только чтение

```
