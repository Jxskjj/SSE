## / - корневой каталог(root)
Вся система находиться в корневом каталоге. Этот каталог является отправной точкой для других каталогов и файлов в системе. После него каталоги и файлы находятся в древовидной системе

## /bin - основные пользовательские двоичные файлы
Этот каталог содержит основные пользовательские двоичные файлы(программы), которые должны присутствовать при монтировании системы в однопользовательском режиме. Не путать с обычными программами, которые скачивает пользователь. Например, firefox будет хранить в каталоге /usr/bin, а такие важные утилиты как bash и тп будут находиться как раз в /bin. Также там хранятся основные команды оболочки такие как ps,ls,ping, grep,cp и другие. Есть и каталог /sbin, но он содержит двоичные файлы для системного администрирования: iptables, reboot, fdisk, ifconfig, swapon.

## /boot - статические загрузочные файлы
Каталог содержит файлы, необходимые для загрузки системы - например загрузчик grub и ядра линукс

## /cdrom - точка монтирования компакт-дисков
Этот каталог не является частью стандарта fns(стандарта файловой системы для Unix систем), но его всё равно можно найти в ос. Это временное место для компакт дисков вставленных в систему. Однако стандартное расположение временных носителей находиться в /media

## /dev - файла устройства 
Линукс представляет устройства в виде файлов и как раз в этом каталоге находятся их представления. Например в каталоге /dev/sda будет находиться наш sata диск и мы можем его разбить на два раздела и файловой системы, чтобы был /dev/sda1 и /dev/sda2. Также в этом каталоге представлены псевдоустройства, которых физически не существует. Например, /dev/radndom выводит рандомные числа 

## /etc - файлы конфигурации
Этот каталог содержит общесистемные файлы конфигурации (например, имя хоста), которые мы можем отредактировать вручную. Пользовательские файлы настроек находятся в домашнем каталоге каждого пользователя

## /home - Домашние папки
Каталог содержит домашнюю папку для каждого пользователя. Например, /home/bob или /home/hdifoe. В папке содержаться файлы пользователя и также его конфигурации. Пользователь может изменять файлы только своей папки, для изменения файлов другого пользователя необходимы root права

## /lib - основные общие библиотеки
Содержит библиотеки, необходимые для основных двоичных файлов из /bin и /sbin. Библиотеки для /usr/bin находятся в /usr/lib

## /lost+found - восстановленные файлы
В случае сбоя файловой системы проверка файловой системы будет выполнена при следующей загрузке  и любые найденные повреждения будут помещены в этот каталог

## /media - съемный носитель
Содержит подкаталоги, в которые монтируются съемные носители, вставленные в пк

## /mnt - временные точки монтирования
Точка монтировки временных файловых систем. Например, /mnt/windows

## /opt - дополнительные пакеты
Содержит подкаталоги для дополнительных пакетов программного обеспечения. Обычно используется проприетарными программами, которые не подчиняются стандартной иерархии файловой системы: /opt/application

## /proc - файлы ядра и процессов
Содержит специальные файлы, которые представляют информацию о системе или процессе. Например там можно найти: /proc/cpuinfo , /proc/{pid}(id процесса), /proc/uptime и другое

## /root - корневой домашний каталог
Домашний каталог рут пользователя, но вместо нахождения в каталоге home а находится в корне

## /run - файлы состояния приложения
Хранит временные файлы запущенных приложений. Они находятся в этом каталоге а не в tmp, тк там они могут быть удалены

## /srv - сервисные данные
Содержит данные об услугах, предоставляемых системой. Например, если бы мы использовали http-сервер apache, то сохранили бы файлы сайта веб-сайта в каталоге внутри /srv

## /tmp - временные файлы
Приложения хранят тут свои временные  файлы, которые удаляются при перезагрузке системы или могут быть удалены при помощи такой утилиты как tmpwatch

## /usr - пользовательские двоичные файлы и данные только для чтения
Хранит второстепенные приложения и файлы, используемые на уровне пользователя а не системы. Например, firefox , vs code и тд

## /var - файлы переменных данных
Место, где программы хранят информацию о времени выполнения, такую как системный журнал, отслеживание пользователей, кеши и другие файлы, которые системы программы создают и управляют. Там можно  найти логи, электронные письма, очереди печати, файлы блокировки и временные файлы, необходимы при перезагрузке
