Для поиска файлов/директорий используется команда find, а для поиска внутреннего содержания команда grep.

Grep и find используют регулярные выражения в некоторых случаях, поэтому критически важно понимать их синтаксис, поэтому начнем с этого.

## Regular Expressions
- `^` : поиск строки начинающего с данного шаблона. Например:
	- `^import` : поиск строки начинающегося с import
	- `^//` : поиск строки начинающегося с коментария
- `$` : поиск строки которая заканчиается шаблоном. Наример:
	- `;$` : оканчивается `;`
	- `!$` : оканчивается на `!`
- `.` : точка обозначает любой ЕДИНСТВЕННЫЙ символ, включая пробел. Например:
	- `l.p` : подойдут строки lsp, lap, l-p и тд
- `[]` : набор любых символов, указанных в скобках. Например:
	- `[a-zA-Z]`: стоит любой ОДИН символ из набора, то есть любая латинская буква
	- `[^0-9]` : `^` в скобках обозначает отрицание. То есть выражение обозначает поиск любого символа кроме цифр
- `*` : ноль или больше повторений. Например:
	- `go*gle` : подойдет `ggle`, `gogle`, `google`, `gooogle`
	- `[0-9]*` : любая комбинация цифр или пустая строка
	- `.*` : любое количество любых символов. Можно использовать как `printf(.*);`, это выведет весь текст внутри скобок

Теперь когда разобраны регулряные выражения можно перейти к самим командам. Начнем с команды `grep`.

## Grep - поиск содержимого по строчкам
команда grep имет следующий синтаксис:
```bash
grep [OPTIONS] PATTERN [FILE...]
grep "hello" file.txt #простой поиск слова в файле
```
Паттерном зачастую служат разобранные ранее регулярные выражения. Файлов является явное его указание или же директория(указывается специальная опция для просмотра всех файлов из директории).
Рассмотрим опции:
- `-i` : поиск без учёта регистра
- `-r` : поиск внутри директории. вместо файла указывается директоия и `grep` просматривает все файла
- `-v` : инверсия условия
- `-w` : поиск целого слова, а не его начала или же конца
- `-E` : использование регулярных выражений
- `l` : вывести только имена файлов, содержащих совпадения

## Find - поиск файлов/директорий
Команда `find` по умолчанию работает рекурсивно, то есть ищет файлы не только в указанной, но и в дочерних директориях. Команда  обаладает уже более сложным синтаксисом и включает в себя не только поиск, но и выполнение действий. Синтаксис:
```bash
find [path/to/search] [criteria] [action]
```
В качестве пути чаще всего указывается `.` (текущая папка) или `/` (вся система).
Основные критерии поиска (Expression):
- `-name` : поиск по имени файла (с учетом регистра). Можно использовать кавычки и звездочку: `"*.txt"`.
- `-iname` : поиск по имени без учета регистра.
- `-type` : тип объекта. `f` — файл, `d` — директория, `l` — ссылка.
- `-size` : поиск по размеру. Например, `+100M` (больше 100 МБ) или `-10k` (меньше 10 КБ).
- `-mtime` : поиск по времени изменения в днях. `-1` — изменен за последние сутки, `+7` — изменен более недели назад.
- `-user` : поиск файлов, принадлежащих конкретному пользователю.
- `-perm` : поиск по правам доступа (например, `-perm 777`).

Опции управления:
- `-maxdepth` : ограничивает глубину поиска в подпапках (например, `-maxdepth 1` искать только в указанной директории).
- `-empty` : искать только пустые файлы или папки.
- `-not` : инверсия условия (искать всё, что НЕ подходит под критерий).

Действия с результатами:
- `-print` : просто выводит список путей, работает по умолчанию даже без указания в поле действия
- `-delete` : удаляет найденные объекты 
- `-exec` : выполняет команду над каждым найденным объектом.
```bash
# Найти все .log файлы и удалить их
find ./logs -type f -name "*.log" -exec rm {} \;
#{} обозначает имя текущего файла, а \; обязательный конец строки 
```
