Си это компилируемый язык, и его компиляция делится на 4 основных этапов. Разеберем каждый из этапов компиляции, а также рассмотрим флаги для комплиятора `gcc`, которые могут дать больше понимания происходящего.

**Этапы компиляции**:
1) Препроцессинг: во время этого этапа исходный код обрабатывается текстовым препроцессором, который удаляет комментарии в коде, раскрывает заголовочные файлы и выполняет макроподстановки.  Результатом выполнения будет файл с раширением `.i`, который является "раздутым" исходным кодом на си. Узнать результат выполнения можно по команде `gcc -E main.c -o main.i`
2) Компиляция: на этом этапе проверяется логика и синтаксис программы и оптимизация кода. В резултате создается код на асемблере, специфичный для процессора. Файл можно создать по команде `gcc -S main.i -o main.s`, где мы получим файл main.c
3) Ассемблирование: тут уже ассемблер переводится в машиный код, который состоин из нулей и единиц. Используется команда `gcc -c main.s -o main.o`
4) Компановка(или же линковка): Линковщик собирает все объектные файлы и библиотеки (например, стандартную библиотеку Си для функции `printf`) в один готовый исполняемый файл. Используется команда `gcc main.c -o main`, а результатом является запускаемый файл в зависимости от системы. В линукс будет файл без расширения. а виндовс `exe` файл

**Флаги компиляции**:
- `-Wall` : включает большинство предупреждений о подозрительном коде(например о неиспользуемой переменной)
- `-Wextra` : включает дополнительные предупреждения, которые не вошли в `-Wall`
- `-O2` : перпеписывает код для средней оптимизации, чтобы код работал быстрее но при этой сохранил разумное время компиляции
- `-g` : добавляет в исполняемый файл отладочную информацию, чтобы можно было использовать отладчик и видеть названия переменных при поиске багов
- `-std=c11` : устанавливает стандарт версии языка, для гарантии работы современных функций
